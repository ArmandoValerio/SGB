Análisis Técnico: Web, Software y Arquitectura

Por Armando Valerio salmeron

Este documento detalla las diferencias entre páginas y aplicaciones web, explora ejemplos del mercado real, analiza el propósito del software y desglosa la arquitectura moderna de desarrollo. Además, incluye un análisis comparativo de plataformas de inventario.

2.- Ejemplos reales de aplicaciones web profesionales

Ejemplo 1.- Canva
Esta aplicación web es de las más conocidas y usadas por usuarios de internet, caracterizada por su principal funcionalidad la cual consiste en crear presentaciones, banners y distintos materiales publicitarios con el estilo que te imagines.

Ejemplo 2.- Spotify
Esta aplicación es de las más populares en el rubro de los reproductores de música, principalmente porque, da acceso una gran variedad de música y su capacidad para crear listas de tus canciones favoritas como también recomendarte canciones tomando en cuenta lo que más escuchas.

3.- ¿Qué tipo de problemas se resuelven con software?

Las problemáticas que se resuelven con software son principalmente las necesidades para compartir información de manera segura y a distancia, de igual manera, se hace para reducir el uso de recursos naturales como lo sería el papel. Así mismo para optimizar los procesos de gestión en general.

4.- Arquitectura general de aplicaciones web

La arquitectura de una aplicación web es el esqueleto que organiza la comunicación entre el usuario y el centro de procesamiento. Este diseño se basa en la segmentación de funciones para asegurar que la interfaz, los procesos lógicos y el almacenamiento operen de forma armónica y eficiente.

~Interfaz de Usuario (Frontend)
Es el entorno digital con el que el usuario final tiene contacto directo a través de su navegador. Su función principal es transformar el código (HTML, CSS, JS) en una experiencia visual y funcional, actuando como el punto de origen de todas las peticiones hacia el sistema central.

~Procesamiento Central (Backend)
Se encarga de recibir, interpretar y responder a las demandas del cliente. Generalmente se organiza en tres niveles de responsabilidad:
•	Nivel de Exposición: Gestiona los puntos de entrada (endpoints) y se encarga de recibir las solicitudes para darles una respuesta adecuada.
•	Nivel Funcional (Lógica): Es el "cerebro" del software donde se ejecutan las reglas operativas, cálculos y validaciones que definen el comportamiento de la app.
•	Nivel de Acceso a Información: Actúa como puente entre la lógica del programa y el almacenamiento físico, facilitando la lectura o escritura de datos.
~Sistema de Almacenamiento (Base de Datos)
Es el depósito donde reside toda la información de manera estructurada y permanente. Ya sea mediante sistemas relacionales (SQL) o no relacionales (NoSQL), garantiza que los datos estén disponibles cada vez que el servidor los requiera.

5.- Plataformas reales similares

~ Ragic!
Ragic es una plataforma sin código (no-code) que permite a empresas y usuarios crear bases de datos y aplicaciones personalizadas de forma intuitiva, similar a usar hojas de cálculo como Excel, pero con la potencia de un sistema de gestión de datos, facilitando la digitalización de procesos empresariales sin necesidad de programar. Permite diseñar formularios, relacionar datos entre diferentes "hojas" (tablas), automatizar flujos de trabajo y gestionar información como inventarios, clientes o proyectos.

~ Ex Libris Alma
Ex Libris Alma es una plataforma de servicios de biblioteca (LSP) integral, basada en la nube y accesible vía web. Está diseñada para unificar y optimizar la gestión de todos los tipos de recursos (impresos, electrónicos y digitales) en un solo sistema, siendo especialmente utilizada por bibliotecas académicas y de investigación.
