## Página Web o Aplicación Web

Las páginas se diseñan principalmente para ofrecer contenido e información estática o  
mínimamente interactivas, mientras que las aplicaciones web se crean para realizar  
tareas más complejas. Ofrecen funciones dinámicas, procesan las entradas del usuario y  
a menudo requieren sistemas back-end para la funcionalidad en tiempo real.

Un simple sitio web de un restaurante puede convertirse en una aplicación web integrando  
reservas online, actualizaciones del menú en tiempo real y funciones de seguimiento de  
pedidos.

## Arquitectura general de aplicaciones web

### Frontend

HTML para estructura, CSS para estilos y JavaScript para interactividad dinámica. Frameworks  
como React, Vue.js o Angular optimizan el desarrollo de interfaces responsive y single-page  
applications.

### Backend

Maneja la lógica del servidor, procesamiento de datos, autenticación y APIs (REST o GraphQL),  
utilizando lenguajes como Node.js, Python (Django/Flask), Java o PHP. Incluye bases de datos  
relacionales (PostgreSQL, MySQL) o NoSQL (MongoDB) y arquitecturas como monolítica,  
microservicios o serverless.

### Entornos

Comprenden desarrollo (local), staging (pruebas) y producción (usuarios finales), gestionados en  
la nube con AWS, Azure o Vercel/Railway. Patrones como Backend for Frontend (BFF) adaptan el  
backend a necesidades específicas de clientes web/móviles, mejorando escalabilidad y  
rendimiento.

## 2 plataformas reales similares a tu idea

### Sistema de Inventarios Mineduc (Chile)

Plataforma gratuita en línea para escuelas subvencionadas. Soporta escaneo de códigos  
de barras (lector o celular), carga masiva vía Excel, inventario de libros/multimedia y  
acceso web con clave SIGE. Ideal para entornos educativos con recursos limitados.

### Timly

Solución de inventario en general. Escaneo QR/códigos desde móvil, CRUD de activos,  
asignaciones y exportación. Enfocado en reducir pérdidas administrativas.
