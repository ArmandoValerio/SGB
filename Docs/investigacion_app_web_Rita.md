# Investigación: Aplicaciones Web

## Diferencia entre Página Web y Aplicación Web

### Página Web
Una página web es un sitio cuyo propósito principal es mostrar información al usuario. Generalmente su contenido es estático o cambia con poca frecuencia y no requiere interacción compleja.

**Características:**
- Contenido informativo  
- Poca o nula lógica de negocio  
- No maneja procesos complejos  
- No suele requerir autenticación  
- Interacción limitada (formularios simples, enlaces)

**Ejemplos:**
- Sitios institucionales  
- Blogs  

### Aplicación Web
Una aplicación web es un sistema interactivo que permite a los usuarios realizar acciones, gestionar información y ejecutar procesos, apoyándose en una base de datos y lógica de negocio.

**Características:**
- Interacción constante con el usuario  
- Manejo de datos dinámicos  
- Autenticación y roles  
- Procesos automatizados  
- Escalable y mantenible  

**Ejemplos:**
- Sistemas de inventario  
- Plataformas educativas  

## Ejemplos Reales de Aplicaciones Web Profesionales

- **Google Drive:** gestión y edición de archivos en tiempo real.  
- **Trello:** administración de tareas y proyectos.  
- **Amazon:** comercio electrónico con inventario, pagos y gestión de usuarios.  

Estas aplicaciones destacan por:
- Manejo eficiente de grandes volúmenes de datos  
- Interacción multiusuario  
- Escalabilidad  
- Acceso desde cualquier navegador  

## ¿Qué Tipo de Problemas se Resuelven con Software?

El software se utiliza principalmente para resolver problemas repetitivos, complejos o propensos a errores humanos.

**Tipos de problemas:**
- **Gestión de información:** inventarios, registros, bases de datos  
- **Automatización de procesos:** préstamos, reportes, validaciones  
- **Optimización del tiempo:** reducción de tareas manuales  
- **Reducción de errores:** evitar duplicados o pérdida de información  
- **Escalabilidad:** crecer sin rehacer el sistema  

## Arquitectura General de una Aplicación Web

Una aplicación web moderna se estructura en capas bien definidas.

### Frontend
Es la capa visible para el usuario.

**Funciones:**
- Interfaz gráfica  
- Formularios  
- Visualización de datos  
- Interacción con el usuario  

**Tecnologías comunes:**
- HTML, CSS, JavaScript  
- Frameworks como React o Angular  

### Backend
Es el núcleo del sistema donde vive la lógica de negocio.

**Funciones:**
- Procesar solicitudes  
- Validar datos  
- Aplicar reglas  
- Comunicación con la base de datos  

**Tecnologías comunes:**
- Node.js, Java, Python, PHP  
- APIs REST  

### Base de Datos
Almacena toda la información del sistema.

**Ejemplos de datos:**
- Libros  
- Usuarios  
- Préstamos  

**Tipos:**
- Relacionales (MySQL, PostgreSQL)  
- No relacionales (MongoDB)  

### Entornos
- **Desarrollo:** donde se construye el sistema  
- **Pruebas:** donde se valida antes de liberar  
- **Producción:** sistema en uso real  

Esta separación garantiza estabilidad y escalabilidad.

## Análisis de 2 Plataformas Reales Similares

### 1: Koha (Sistema de Gestión Bibliotecaria)

**Descripción:**  
Sistema web open-source utilizado en bibliotecas para administrar catálogos, usuarios y préstamos.

**Funciones:**
- Catálogo de libros  
- Gestión de usuarios  
- Control de préstamos  

**Limitaciones:**
- Interfaz compleja  
- Curva de aprendizaje alta  
- Poco amigable para bibliotecas pequeñas  

### 2: LibraryThing

**Descripción:**  
Plataforma web para catalogar colecciones de libros.

**Funciones:**
- Registro de libros  
- Búsquedas  
- Organización por categorías  

**Limitaciones:**
- No está pensada para inventarios institucionales  
- No maneja flujos administrativos completos
